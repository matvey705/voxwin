# VoxWin — локальная голосовая диктовка для Windows

Аналог VoxLocal (macOS) и Whisper Flow, но для Windows, полностью локальный и
бесплатный: зажали клавишу → сказали → текст появился в активном поле любого
приложения. Ничего не уходит в облако, подписок нет.

## Возможности

- **Локальное распознавание** — faster-whisper (CTranslate2): модели
  `tiny … large-v3`, включая `large-v3-turbo`; INT8/FP16-квантизация.
- **CPU и GPU** — автоопределение CUDA c откатом на CPU int8; DirectML не
  требуется, на CPU работает из коробки.
- **Два режима записи одновременно**: PTT (удерживайте `F9`) и старт/стоп
  (`F10`). Клавиши настраиваются, включая комбинации (`ctrl+alt+space`).
- **Вставка в любое приложение** — через буфер обмена (Ctrl+V, с
  восстановлением прежнего содержимого) или посимвольный Unicode-ввод
  (для терминалов и «капризных» полей).
- **Silero VAD** — тишина отсекается до распознавания → быстрее и точнее.
- **Пост-обработка**: авто-заглавная, чистка пробелов/пунктуации, удаление
  слов-паразитов (отключено по умолчанию), **словарь замен** («джира» → Jira)
  и термины-подсказки, которые Whisper видит как initial prompt.
- **Опционально — доработка текста локальной LLM** через Ollama
  (по умолчанию 127.0.0.1, как в VoxLocal; адрес настраивается — указывайте
  только доверенный локальный хост, туда уходит текст диктовки).
- **Трей + оверлей**: статус (готов/запись/распознаю), живой уровень
  микрофона, история последних фраз (копирование в один клик).
- Авто-язык или фиксированный (русский, английский и ещё 13).
- Автозапуск при входе в Windows, звуковые сигналы, лимит записи,
  автостоп при отключении микрофона.

## Установка (из исходников)

Нужен Python 3.10–3.12 (64-bit). В папке проекта:

```powershell
py -m venv .venv
.venv\Scripts\python -m pip install --upgrade pip
.venv\Scripts\python -m pip install -r requirements.txt
```

GPU-ускорение (NVIDIA, драйвер установлен — сами CUDA/cuDNN ставить не надо,
они приедут pip-пакетами):

```powershell
.venv\Scripts\python -m pip install -r requirements-gpu.txt
```

Запуск:

```powershell
.venv\Scripts\python -m voxwin          # с консолью (видно логи)
VoxWin.bat                              # без консоли (pythonw)
.venv\Scripts\python -m voxwin --diagnose        # проверка окружения
.venv\Scripts\python -m voxwin --diagnose-full   # + тест загрузки модели
```

При первом распознавании модель скачается с Hugging Face в
`%LOCALAPPDATA%\VoxWin\models` (small ≈ 480 МБ) — один раз.

## Использование

| Действие | Как |
|---|---|
| Продиктовать фразу | Удерживайте `F9`, говорите, отпустите |
| Длинная диктовка | `F10` — старт, `F10` ещё раз — стоп |
| Отменить запись | Трей → «Отменить запись» |
| Настройки | Правый клик по иконке в трее → «Настройки…» |
| Скопировать прошлую фразу | Трей → «История» → клик по фразе (кладёт в буфер) |

Текст вставляется туда, где стоит курсор — фокус приложения не трогается.

## Выбор модели (RTX 3050 Ti 4 ГБ / средний CPU)

| Модель | VRAM/RAM | Скорость | Когда брать |
|---|---|---|---|
| tiny/base | ~0.3–0.5 ГБ | мгновенно | команды, короткие фразы |
| **small** | ~1 ГБ | ~0.3–1 с на фразу (GPU) | **по умолчанию, лучший баланс** |
| medium | ~2.5 ГБ | 1–3 с | сложная терминология |
| large-v3-turbo | ~1.8 ГБ | ~как small | максимум качества за свою цену |
| large-v3 | ~4.5 ГБ | медленно на 4 ГБ | лучше не надо на этой карте |

Латентность дополнительно снижают: beam size 1–3, включённый VAD,
«Загружать модель при старте».

## Сборка дистрибутива

```powershell
.venv\Scripts\python -m pip install pyinstaller
.venv\Scripts\python -m PyInstaller packaging\voxwin.spec --noconfirm
# → dist\VoxWin\VoxWin.exe
```

Инсталлятор: открыть `packaging\installer.iss` в
[Inno Setup](https://jrsoftware.org/isinfo.php) → Compile →
`packaging\Output\VoxWin-Setup.exe`.

## Устранение неполадок

- **Модель не скачивается (SSL/прокси).** В requirements уже есть
  `truststore` (VoxWin включает его при старте — сертификаты берутся из
  хранилища Windows, TLS-перехват антивирусом перестаёт мешать). Мёртвый
  системный прокси VoxWin обходит сам: при ошибке скачивания повторяет
  запрос с `NO_PROXY=*`. Для pip за таким же прокси:
  `--trusted-host pypi.org --trusted-host files.pythonhosted.org`.
- **Текст не вставляется в программу, запущенную от администратора.**
  Ограничение Windows (UIPI): запустите VoxWin тоже от администратора.
- **CUDA не подхватилась.** `.venv\Scripts\python -m voxwin --diagnose`
  покажет число CUDA-устройств; поставьте `requirements-gpu.txt` и обновите
  драйвер. При любой ошибке CUDA приложение само откатывается на CPU.
- **Хоткей не срабатывает в игре/эмуляторе.** Некоторые полноэкранные
  приложения перехватывают ввод раньше глобального хука — смените клавишу
  или запустите VoxWin от администратора.
- **Буфер обмена.** VoxWin восстанавливает текст, HTML/RTF, картинки
  (CF_DIB) и скопированные файлы (CF_HDROP). Экзотические форматы, которые
  восстановить нельзя, VoxWin не трогает вовсе (пропускает восстановление).
  Если приложение медленно обрабатывает Ctrl+V и вставляет «старый» буфер —
  увеличьте «Задержку восстановления» в настройках вставки.
- **Логи**: `%APPDATA%\VoxWin\logs\voxwin.log`, конфиг —
  `%APPDATA%\VoxWin\config.json`, сброс:
  `.venv\Scripts\python -m voxwin --reset-config`.

## Архитектура

```
клавиатурный хук (keyboard)          PortAudio callback (sounddevice)
        │  сигналы Qt                        │ float32 16kHz chunks
        ▼                                    ▼
   VoxApp (Qt main thread) ◄──────────── Recorder
   состояние: idle/recording/processing
        │ submit(audio)
        ▼
   Transcriber worker thread
   faster-whisper (CTranslate2, CUDA→CPU fallback, Silero VAD)
        │ текст
        ▼
   PostProcessor (словарь, паразиты, регистр) → [Ollama refine]
        │
        ▼
   Injector: снапшот буфера → Ctrl+V (SendInput) → восстановление
        │ сигнал sigDone
        ▼
   Tray/Overlay: статус, история, уведомления
```

Приватность: аудио живёт только в оперативной памяти, на диск не пишется;
в логи попадают только первые 120 символов распознанного текста
(отключается уровнем логирования).
